¥begin{abstract}
In this project, we develop a system that classifies speech into 22 Indian languages using the badrex/nnti-dataset-full dataset. A major challenge is
speaker bias since each language only has five speakers. We first reproduce the baseline model and apply different pre-trained speech models and
hyperparameter tuning to improve performance. To reduce speaker bias, we use data-centric augmentation and model-based adversarial training (DANN). 
We then evaluate multiple speaker bias mitigation variants and find that DANN gives the best result: 0.5576 accuracy and 0.5426 macro-F1. 
We also analyze bias source, confusion patterns, and last-layer t-SNE behavior. 
Results show clear bias reduction and improvement in weak-class performance, but speaker information is not completely removed.
¥end{abstract}

¥section*{1 Introduction}
This project focuses on language identification with significant speaker variation. The main challenge is learning language features from a small number of training speakers while generalizing to unseen validation speakers.

Implementation uses PyTorch ¥citep{paszke2019pytorch}, Hugging Face Transformers/Datasets ¥citep{wolf2020transformers}, the NNTI dataset card ¥citep{badr2026dataset}, MMS resources ¥citep{pratap2023mms,facebookmmsmodelcard}, PyTorch Audio augmentation practice ¥citep{pytorchaudioaugtutorial}, scikit-learn ¥citep{pedregosa2011scikit}, Matplotlib ¥citep{hunter2007matplotlib}, wav2vec2 ¥citep{baevski2020wav2vec2}, HuBERT ¥citep{hsu2021hubert}, Domain-Adversarial Neural Network (DANN) ¥citep{ganin2014dann}, and t-SNE ¥citep{maaten2008tsne}.

¥section*{2 Dataset}
We use ¥texttt{badrex/nnti-dataset-full} ¥citep{badr2026dataset}. Core split statistics:
¥begin{itemize}
    ¥item Languages: 22
    ¥item Train samples: 8689
    ¥item Validation samples: 3300
    ¥item Train unique speakers: 110
    ¥item Validation unique speakers: 681
    ¥item Train/validation speaker overlap: 0
¥end{itemize}

The validation data comes from speakers that are different from those in the training data.

¥section*{3 Experiments}
¥subsection*{3.1 Experimental Setup}
To keep comparisons fair, we use the same dataset split and fixed random seed and changed only intended components per run. Shared protocol:
¥begin{itemize}
    ¥item Fixed split: same train/validation files across runs.
    ¥item Fixed seed: 42.
    ¥item Best-checkpoint loading enabled ¥texttt{load¥_best¥_model¥_at¥_end=True}.
    ¥item Same diagnostics family: per-language accuracy, confusion matrix, speaker diagnostics.
¥end{itemize}

¥subsection*{3.2 Evaluation Metrics}
We report:
¥begin{itemize}
    ¥item Global metrics: validation accuracy, macro-F1, eval loss.
    ¥item Class-level metric: per-language validation accuracy.
    ¥item Error structure: off-diagonal confusion pairs from confusion matrix.
    ¥item Speaker diagnostics: overlap count and unseen-speaker accuracy.
    ¥item Representation diagnostics: last-layer t-SNE (where artifacts are available).
¥end{itemize}

Macro-F1 is prioritized over accuracy for evaluating speaker-bias mitigation.

¥section*{4 Baseline Reproduction and Improvements}
¥subsection*{4.1 Methods}
The baseline-improvement pipeline follows two stages:
¥begin{itemize}
    ¥item ¥textbf{Baseline reproduction}: reproduce the baseline by fine-tuning the ¥texttt{facebook/mms-300m} model using the provided starter script, aiming to reach the 31.96¥% top-1 accuracy
    
    ¥item ¥textbf{Improvement stage}: we experiment with other pretrained speech models and better hyperparameter tuning (e.g., learning rate, batch size, warmup steps, and training schedules). This results in a tuned mHuBERT model used as the reference for later experiments.
¥end{itemize}

To exceed the baseline performance, we increased the number of training epochs, enabled dropout, set the learning rate =  ¥(3¥times10^{-5}¥), and used (¥texttt{group¥_by¥_length=True})

The main hyperparameter settings reported for this baseline-improvement stage are: effective batch control (per-device batch size around 16 with gradient accumulation), checkpointing policy (¥texttt{load¥_best¥_model¥_at¥_end=True}) and model-selection metric (accuracy for the tuned no-mitigation reference) .
These training settings significantly affect the final model performance.

¥section*{5 Speaker-Bias Mitigation}
¥subsection*{5.1 Methods}
We used both data-centric and model-based mitigation. We also tested several intermediate variants (Mitigation 2, 3A, 3B, and 4) to understand the trade-offs.

¥textbf{Data-centric Augmentation Mitigation:}
¥begin{itemize}
    ¥item Mitigation 1: speed/noise ($p=0.30$, speed 0.97--1.03, noise std 0.0005--0.002).
    ¥item Mitigation 2: gentler speed/noise ($p=0.25$, speed 0.98--1.02, noise std 0.0005--0.0015) with macro-F1 checkpointing.
    ¥item Mitigation 3A: revert to M1 augmentation strength with macro-F1 checkpointing and more frequent eval/save steps (100/100).
    ¥item Mitigation 3B: milder fallback from M3A ($p=0.28$, speed 0.975--1.025, noise std max 0.0018). 
    ¥item Mitigation 4: all data-centric augmentation (speed/noise + pitch shift + spectral masking) with global augmentation gate and t-SNE export.
¥end{itemize}

¥textbf{Model-based Mitigation (DANN):}
¥begin{itemize}
    ¥item Architecture: shared encoder with language and adversarial speaker heads.
    ¥item Training logic: a gradient reversal layer forces the encoder to reduce speaker-identifiable information while preserving language-discriminative features.
    ¥item Final settings: ¥texttt{speaker¥_loss¥_weight=0.1}, ¥texttt{grl¥_lambda=1.0}, lambda schedule enabled, speaker-head dropout 0.1.
    ¥item Run design: augmentation was disabled in the final DANN run to isolate the effect of adversarial mitigation from augmentation effects.
¥end{itemize}

Encoder objective:
¥[
¥mathcal{L}_{enc} = ¥mathcal{L}_{lang} - ¥lambda ¥mathcal{L}_{speaker}.
¥]

¥subsection*{5.3 Results}
Overall, DANN is the strongest mitigation result: +0.0306 accuracy and +0.0678 macro-F1 over tuned mHuBERT model (no-mitigation), and +0.0188 accuracy and +0.0558 macro-F1 over Mitigation 1. Data-centric methods were useful but sensitive to augmentation strength. Mitigation 1 was the best data-centric variant, while mixed augmentation (Mitigation 4) underperformed.

Confusion behavior also improved for several commonly confused language pairs. (Table‾¥ref{tab:mitigation_confusions}), especially ¥texttt{sindhi$¥rightarrow$punjabi} and ¥texttt{hindi$¥rightarrow$urdu}. Some pairs such as ¥texttt{konkani$¥rightarrow$marathi} are still difficult, so the bias is reduced but not completely removed.

Speaker-bias diagnostics show that these improvements are not caused by speaker leakage. The training and validation sets have no overlapping speakers (¥texttt{overlap¥_speaker¥_count=0}). 
Likewise, unseen-speaker accuracy increases from 0.4676 (untuned) to 0.5576 (DANN). The weak-class mean accuracy increased from 0.0227 to 0.3547, indicating an improvement in performance on previously weak classes. 

¥section*{6 Analysis}
¥subsection*{6.1 Bias Source and Learning Challenges}
The bias source in this dataset is the limited speaker diversity in training data. The training set contains 110 speakers over 22 languages while the validation set contains 681 speakers with 3300 samples, and all validation speakers are unseen during training. 
This creates a speaker bias problem because the model may learn speaker-specific characteristics such as (voice quality or speaking style) instead of learning the language itself.

Learning from this biased data is difficult for two reasons. ¥textbf{First}, with few speakers per language, the encoder can partially fit speaker characteristics during training even though the validation speakers are different. 
¥textbf{Second}, it is harder for the model to accurately distinguish between related languages because many of them have similar prosodic and acoustic patterns. As a result, some languages remain weakly separated in the learned representation.

Evidence from our experiments:
¥begin{itemize}
    ¥item Untuned reference has near-zero classes (for example maithili 0.0000, konkani 0.0000, sindhi 0.0333).
    ¥item Confusion matrix shows strong confusion between certain language pairs (for example konkani$¥rightarrow$marathi and sindhi$¥rightarrow$punjabi).
    ¥item Validation is fully unseen-speaker (seen-speaker samples = 0, unseen-speaker samples = 3300), so weak performance cannot be explained by train/validation speaker overlap.
¥end{itemize}


¥subsection*{6.2 Mitigation Approach and Results}
Our approach combines data-centric mitigation and model-based adversarial de-biasing to reduce speaker bias. 
¥begin{itemize}
    ¥item Data-centric mitigation (especially Mitigation 1) improves robustness by breaking fixed speaker-style dependence.
    ¥item DANN directly penalizes speaker-identifiable features in the shared encoder.
¥end{itemize}

The results indicate strong but partial improvement. The overall results are shown in Table‾¥ref{tab:dann_impact}.

¥subsection*{6.3 Confusion Patterns}
Most frequent confused languages are:
¥begin{itemize}
    ¥item ¥texttt{konkani$¥rightarrow$marathi}
    ¥item ¥texttt{sindhi$¥rightarrow$punjabi}
    ¥item ¥texttt{hindi$¥rightarrow$urdu}
¥end{itemize}

These confusions occur mainly because the languages share similar linguistic and acoustic characteristics. 
As shown in Table‾¥ref{tab:mitigation_confusions}, DANN reduces dominant confusions, but not all.

Pattern-specific interpretation:
¥begin{itemize}
    ¥item ¥texttt{hindi$¥rightarrow$urdu}: similar words and pronounciation patterns in conversational speech.
    ¥item ¥texttt{konkani$¥rightarrow$marathi}: regional similarity and overlapping speaking styles.
    ¥item ¥texttt{sindhi$¥rightarrow$punjabi}: difficult boundary under uneven class difficulty
    ¥item $¥texttt{odia}¥rightarrow¥texttt{bengali}$ and $¥texttt{santali}¥rightarrow¥texttt{bengali}$: neighboring-language proximity
¥end{itemize}

The results also show that DANN reduces dominant confusions compared to the untuned mHuBERT model including (¥texttt{sindhi$¥rightarrow$punjabi}, ¥texttt{santali$¥rightarrow$bengali}, ¥texttt{odia$¥rightarrow$bengali}, ¥texttt{hindi$¥rightarrow$urdu}, ¥texttt{konkani$¥rightarrow$marathi}). 
Consequently, some alternative confusions increase (for example ¥texttt{bodo$¥rightarrow$manipuri} and ¥texttt{nepali$¥rightarrow$manipuri} when compared with Mitigation 1), indicating that the model still struggles with closely related languages.

¥section*{7 Conclusion}
Tuned mHuBERT provides a strong reference model without bias mitigation. The results show that mitigation performance depends on the training setup. Mild data augmentation improves the model, while very strong augmentation can reduce performance. The best final model is DANN, which achieves the highest accuracy and improves performance on weaker languages under unseen-speaker validation. The analysis shows that speaker bias is reduced, but some difficult language confusions and speaker-related signals still remain.
